# QuestLearn Part I Submission Assembly Checklist

This checklist is designed to minimize coordination. Follow it in order and do not skip ahead.

## 1. Final Files to Assemble

Use these files as the source of truth:

- [Part-I-System-Overview.md](./Part-I-System-Overview.md)
- [Use-Cases.md](./Use-Cases.md)
- [ERD-UML.md](./ERD-UML.md)
- [Requirements-Traceability.md](./Requirements-Traceability.md)
- [Gantt-Chart.md](./Gantt-Chart.md)
- [Use-Case-Diagram-Spec.md](./Use-Case-Diagram-Spec.md)
- [Terminology-Sheet.md](./Terminology-Sheet.md)

## 2. Minimum Submission Package for Part I

The final Part I document should contain:

- cover page
- project title and team information
- system overview
- objectives
- scope
- main user scenarios
- innovation summary
- use case diagram
- formal use case descriptions
- process-flow diagrams
- ERD or class diagram
- Gantt chart

## 3. Copy-Paste Assembly Order

### Section A: Title and Intro

- [ ] Paste project title and tagline from [Part-I-System-Overview.md](./Part-I-System-Overview.md).
- [ ] Add team names, tutorial code, group number, and lecturer name manually.

### Section B: System Overview

- [ ] Paste sections `1.1` to `1.8` from [Part-I-System-Overview.md](./Part-I-System-Overview.md).
- [ ] Keep headings consistent.
- [ ] Do not reword unless fixing grammar or shortening repeated lines.

### Section C: Use Case Section

- [ ] Paste the role-based use case list from [Use-Cases.md](./Use-Cases.md).
- [ ] Draw the UML use case diagram using [Use-Case-Diagram-Spec.md](./Use-Case-Diagram-Spec.md).
- [ ] Paste the formal use case descriptions from [Use-Cases.md](./Use-Cases.md).
- [ ] Convert the selected process-flow drafts into clean diagrams and insert them under the use case section.

### Section D: ERD and Class Support

- [ ] Paste the core entities and main relationships summary from [ERD-UML.md](./ERD-UML.md).
- [ ] Redraw the ERD cleanly in a diagram tool.
- [ ] If required by the lecturer, also redraw the expanded class diagram from [ERD-UML.md](./ERD-UML.md).

### Section E: Project Planning

- [ ] Redraw the Gantt chart from [Gantt-Chart.md](./Gantt-Chart.md).
- [ ] Insert the Gantt chart image or exported PDF snapshot into the report.

### Section F: Final Quality Control

- [ ] Use [Terminology-Sheet.md](./Terminology-Sheet.md) to standardize terms.
- [ ] Use [Requirements-Traceability.md](./Requirements-Traceability.md) to confirm TT7L coverage.

## 4. Diagram Set to Produce

To keep workload minimal, produce this exact set:

- [ ] 1 UML use case diagram
- [ ] 4 to 6 process-flow diagrams
- [ ] 1 ERD
- [ ] 1 class diagram only if your lecturer expects both
- [ ] 1 Gantt chart

If time is tight:

- prioritize use case diagram, ERD, and Gantt chart first
- keep class diagram only if it clearly adds marks or is explicitly requested

## 5. Who Should Do What Right Now

### Team Lead

- [ ] merge all text into one master Part I document
- [ ] decide whether to include the class diagram in addition to the ERD
- [ ] do the final rubric check
- [ ] do the final export and file naming

### Member B

- [ ] redraw ERD cleanly
- [ ] redraw class diagram if included
- [ ] verify no use case depends on a missing entity

### Member C

- [ ] redraw use case diagram
- [ ] redraw process flows
- [ ] redraw Gantt chart
- [ ] clean the document formatting

### Member D

- [ ] run the checklist in [Requirements-Traceability.md](./Requirements-Traceability.md)
- [ ] check headings, captions, labels, and term consistency
- [ ] confirm missing TT7L items are no longer missing in the merged report

## 6. Final Red-Team Check Before Export

- [ ] Email verification appears in overview, use cases, and at least one process flow.
- [ ] Assignment management appears in overview, use cases, and ERD.
- [ ] Activity tracking appears in overview, use cases, and ERD.
- [ ] Academic Advisor is named consistently everywhere.
- [ ] The use case diagram is a real UML diagram, not a flowchart.
- [ ] The ERD includes `Assignment`, `AssignmentSubmission`, `Announcement`, and `AdvisorStudentAssignment`.
- [ ] The Gantt chart shows dates, owners, and milestones.
- [ ] The report reads like one document, not separate pasted notes.

## 7. Export Checklist

- [ ] PDF opens correctly
- [ ] diagram text is readable at normal zoom
- [ ] file name follows lecturer format
- [ ] no local-only files such as `team member.md` are uploaded
